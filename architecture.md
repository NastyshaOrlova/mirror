# Архитектура

## Стек

- Бэкенд: Node.js, NestJS, TypeScript, PostgreSQL, TypeORM
- Клиент: определяется после завершения API

## Структура

NestJS-приложение с модульной структурой. Каждый модуль самодостаточен: сущности, сервис, контроллер. Модули взаимодействуют через инъекцию сервисов.

Бэкенд разрабатывается первым как самостоятельный API. Платформа клиента определяется после стабилизации API.

Фоновые процессы (cron): сброс дневных состояний, генерация задач на день, очистка устаревших данных.

### Модули

- **Auth** — регистрация, логин/логаут, JWT (access + refresh), удаление аккаунта
- **Notes** — текстовые записи, фильтрация по дате и диапазону
- **Morning Reminder** — настраиваемое утреннее напоминание, состояние прочитано/нет, авто-сброс
- **Evening Question** — вопрос для рефлексии, история ответов, состояние отвечено/нет
- **Checklist** — повторяющиеся и разовые задачи, генерация на день из шаблонов
- **Export** — собирает данные из Notes и Evening Question в файл за период

### Зависимости

- Auth при регистрации создаёт дефолтные конфиги и стартовые задачи в остальных модулях
- Export читает данные из Notes и Evening Question
- Cron сбрасывает состояния Morning Reminder, Evening Question и генерирует/чистит Checklist

## Архитектурные решения

**Состояния сбрасываются cron-ом раз в сутки.** isRead, isAnswered — это текущее состояние, а не запись за конкретный день. Так проще модель данных, и нет ситуации когда действие случайно относится ко вчера.

**Старые данные чистятся автоматически.** DailyTask и неактивные разовые Task удаляются по расписанию. Статистика не собирается — хранить их незачем.

**При регистрации создаются дефолтные настройки.** Приложение работает сразу, пользователь меняет только если хочет.

**Refresh token обновляется при каждом использовании.** Пока пользуешься — не выкидывает.
